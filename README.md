# 剣道選手の成長促進Webアプリケーション「Kendo Life」

## 概要
Kendo Lifeは、剣道選手が日々の稽古や試合を振り返り、継続的な成長につなげることを目的としたWebアプリケーションです。  試合スコア、得点の流れ、スケジュール、目標などを一元管理することで、記録を「残す」だけでなく「振り返って活用する」ことを目指して開発しました。

## 対象ユーザー
- 日々の稽古や試合を通して成長を目指す剣道選手

## 主な機能
- **スケジュール管理機能**：試合や稽古などの日々の予定をカレンダー上に登録・管理 できる 
- **ToDo・目標管理機能**：日々の課題や目標を整理できる
- **試合スコア記録機能**：剣道のスコアを個人戦・団体戦それぞれ記録し、振り返ることができる 
- **掲示板機能**：剣道に関する質問や情報を投稿し、ユーザー同士で知識や経験を共有できる
- **プロフィール管理機能**：選手情報の登録・編集

## 制作背景
自身が小学校から高校まで剣道を経験する中で、試合スコアを紙で記録する運用に不便さを感じていました。  紙がかさばる、紛失しやすい、過去の記録を振り返りにくいといった課題があり、記録が十分に活用されていない状況でした。そのような剣道の課題を改善したいと思いWebアプリ開発を決意しました。よりよいアプリを作るため、剣道経験のある友人8名にヒアリングを行ったところ、全員が同様の課題を感じていることが分かりました。また、既存の剣道向けアプリを調査した結果、スコア記録に特化したものは小数存在するものの、日々の活動や目標まで一体的に管理できるサービスは少ないと感じ、ヒアリングで出た要望を取り入れ成長を促進することをコンセプトにした本アプリケーションの開発に至りました。

## 工夫した点

### ① 技入力の「一つ戻る」機能
スコア入力時の操作性向上を目的として、直前の操作を取り消せる「一つ戻る」機能を実装しました。  
JavaScriptで入力履歴をactionHistoryという変数として管理し、技・反則などの入力内容を時系列で保持しています。
「一つ戻る」操作時には、最新の履歴を削除し、その内容に応じてスコア表示や状態を更新することで、  
直前の入力のみを安全に取り消せる仕組みとしました。  
これにより、ユーザーが入力操作を行いやすいUIを実現しています。

### ② スコア表のレスポンシブ対応
剣道のスコア表は横長のレイアウトが一般的ですが、そのままスマートフォンで表示すると視認性・操作性が大きく低下します。  
本アプリでは、画面の向きや端末に応じてレイアウトを切り替えるレスポンシブ対応を行いました。
横画面では従来のスコア表に近いレイアウトを、縦画面ではスマートフォン向けに情報を縦に再配置することで、  
端末ごとに使いやすいUIを実現しています。

## 苦労した点

### ① レスポンシブ対応におけるHTML構造の同期
レスポンシブ対応の実装において、当時の知識の範囲で可読性と実装のしやすさを優先し、PC版とスマートフォン版でHTML構造を分けて実装しました。その結果、両レイアウト間で入力内容を同期する必要が生じ、 JavaScriptでデータ連携の処理を実装する必要がありました。  
この経験から、HTML構造を分けずCSSのみで制御する設計の重要性を学び、今後のリファクタリング課題として捉えています。

### ② 剣道特有のルール・記載方法のロジック化
剣道の試合では、反則や得点の扱いなど競技特有のルールが多く、単純な数値管理では正確に表現できません。実際のスコア表の書き方やルールを整理し、条件分岐やデータ構造としてロジック化する点に苦労しました。

例：
- 反則を2回取ると反則マークを消し、相手に一本を追加  
- 最初の一本目には丸を付ける必要がある  
- 個人戦・団体戦を考慮したモデル設計（Matchモデルで一括管理）  

試行錯誤を重ねることで、試合内容を正しく記録・再現できるスコア管理機能を実装しました。
